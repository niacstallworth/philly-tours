# AR Production Sheet

This document translates the current tour catalog into a realistic AR production plan.

## Scope Rule

- Every stop should have content.
- Only selected stops should receive full AR treatment.
- Default rollout target:
  - `10-20%` of stops = Hero AR
  - `30-50%` of stops = Light AR
  - remaining stops = map, audio, photos, text

## AR Types

| AR Type | Description | Best Use | Relative Effort |
| --- | --- | --- | --- |
| Portal Reconstruction | Framed window into a historical scene or lost environment | Demolished sites, changed streetscapes, memorial reconstructions | High |
| Object on Plinth | Single 3D artifact, symbol, trophy, or invention in front of user | Inventors, sports, fraternal symbols, ritual objects | Medium |
| Floating Story Card | Spatial text, portraits, dates, quotes, archival cards | Secondary stops, dense story routes, low-cost AR coverage | Low |
| Before/After Overlay | Visual comparison aligned to current location | Architecture, transit, memorial, corridor tours | Medium |
| Historical Figure Presence | Stylized silhouette or voice-led apparition | Legacy figures, speeches, ceremonial moments | Medium |
| Animated Diagram | 3D explainer of mechanism, system, or process | Medical, science, inventors, transit | Medium |
| Route Ghost | Directional AR breadcrumbs or path line | Corridor tours, campus tours, line-based tours | Low |
| Puzzle Reveal | Tap-to-reveal clue layers, symbols, hidden panels | Scavenger hunts and ritual/gamified tours | Medium |

## Production Principles

- Do not build bespoke 3D scenes for every stop.
- Build reusable AR systems first, then assign them across tours.
- Prioritize stops with:
  - strong visual story
  - safe standing area
  - high cultural significance
  - good coordinate confidence
- Use `Floating Story Card` as the default AR format for secondary stops.
- Reserve `Portal Reconstruction` for the few stops that justify heavier production.

## Recommended First Systems

Build these first:
- Portal Reconstruction
- Object on Plinth
- Floating Story Card

Add later:
- Before/After Overlay
- Historical Figure Presence
- Animated Diagram
- Route Ghost
- Puzzle Reveal

## What This Enables

With this plan, the app can:
- support every tour with meaningful content
- feature AR in every major tour pack
- stay within a realistic production scope
- scale without requiring full-scene AR at every single stop
